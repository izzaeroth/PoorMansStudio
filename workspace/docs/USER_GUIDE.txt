Poor Man's Studio - User Guide
==============================

Poor Man's Studio is a local Windows desktop music workstation for importing, editing, previewing, recording, saving, and rendering music without needing a full commercial DAW.

The app is built around projects, tracks, sequences, and a workspace folder. MIDI and MusicXML material becomes MIDI tracks. Imported or recorded audio becomes AudioClip tracks. MIDI tracks can use SF2/SF3, SFZ, or scanned VST3 plugin instruments.

Open the illustrated PDF guide from Help > Open User Guide PDF. This text guide is the plain-text companion.

Quick Start
===========

1. Choose a project backend in Project Defaults.
2. Choose the matching instrument or library.
3. Click Apply Project Defaults before importing or adding new MIDI tracks.
4. Start from MusicXML, MXL, MIDI, audio, or an existing .mwproj project.
5. Edit track layout, sequences, notes, and per-track instruments.
6. Preview the track, sequence, or full project.
7. Save the project as .mwproj.
8. Render final audio or MIDI output.

Project Defaults affect future imported or newly added MIDI tracks. Existing tracks are not overwritten by Apply Project Defaults; change existing tracks with per-track controls or inside a Piano Roll window.

Main Window Areas
=================

Project Controls
----------------

Use Start From File to import MusicXML, MXL, MIDI, audio, or .mwproj files. Use Open .mwproj to reopen saved projects. Save Project writes the current project metadata to the project folder.

Project Defaults
----------------

The Project Defaults area chooses the backend that new/imported MIDI tracks should use:

- SoundFont/SF3: choose a SoundFont file and preset.
- SFZ: choose an SFZ instrument.
- VST3 Plugin: scan plugins, choose a plugin instrument, and apply the project default.

Use Apply Project Defaults after choosing the backend and instrument. Existing track assignments stay as they are.

Track Controls
--------------

Use Apply Track Settings to commit per-track instrument changes. Use Change Library to choose a library for the selected MIDI track. Use Open VST Plugin when the selected track is assigned to a usable VST3 plugin instrument.

Console And Status
------------------

The console reports scans, imports, render status, warnings, and errors. Helper bubbles are enabled by default and appear after hovering on controls for about two seconds. Toggle them from Help > Helper Bubbles.

Workspace And Project Folders
=============================

Recommended workspace structure:

workspace\
  input\
  exports\
  projects\
  soundfonts\
  fluidsynth\
  ffmpeg\
  sfz\
  vst3\
  sfizz\
  settings\
  temp\
  themes\

Recommended project structure:

workspace\
  projects\
    Project Name\
      Project Name.mwproj
      input\
        midi\
        mxl\
        audio\
          imported\
          recorded\
          temp\
      output\
      renders\

Important rules:

- The .mwproj file stores metadata and project-relative paths.
- Large imported or recorded audio files are not embedded in .mwproj.
- AudioClip media lives in the project folder under input\audio.
- Move or share the whole project folder, not only the .mwproj file.
- Save/open keeps the project folder name, .mwproj filename, project name, and base name aligned when using the default project layout.

Instrument Sources
==================

SoundFont/SF3
-------------

SoundFont and SF3 instruments are rendered through FluidSynth. Choose a file, refresh the preset list, choose a preset, then apply the project or track setting. General MIDI preset matching is used when importing tracks where possible.

SFZ
---

Poor Man's Studio supports individual SFZ files and sidecar SFZ packs. If the selected file is:

workspace\sfz\TheOpenSourceDrumKit-Complete-V1.sfz

then the app also checks this pack root:

workspace\sfz\TheOpenSourceDrumKit-Complete-V1\

Keep the .sfz file and the same-name folder together. The SFZ option list treats Name.sfz plus Name\ as one public pack option. Preflight and render checks resolve sample= and #include paths from the pack root when needed.

VST3 Plugin Instruments
-----------------------

VST3 support is experimental. A plugin can crash inside its own native code, and because instruments are currently hosted in-process, a plugin crash can take down the main app. Save your work before trying unfamiliar plugins.

Poor Man's Studio scans VST3 bundles from:

workspace\vst3
C:\Program Files\Common Files\VST3

A Windows VST3 plugin may look like a file in Explorer, but it is often a bundle folder ending in .vst3. Keep the outer .vst3 bundle intact. The app registers the outer bundle path, not the inner Contents\x86_64-win binary.

Recommended VST3 project workflow:

1. Put test/user VST3 bundles in workspace\vst3 or install plugins in the standard Windows VST3 folder.
2. Use VST Plugins > Scan VST3 Plugins.
3. Set the project Backend dropdown to VST3 Plugin.
4. Choose a scanned VST3 instrument from the Instrument dropdown.
5. Click Apply Project Defaults.
6. A one-time dragon warning appears before VST3 becomes active through project defaults, track assignment, plugin editor opening, preview, or render. Click OK to acknowledge that the feature is experimental and may crash the program.
7. Import MIDI/MusicXML or add new MIDI tracks.

New/imported MIDI tracks seed from the applied VST3 project default. Each seeded track receives its own independent plugin assignment and state, even when multiple tracks use the same plugin bundle.

Important VST3 rules:

- Applying Project Defaults affects future imported/added tracks.
- Existing tracks are not overwritten.
- Use Apply Track Settings to commit per-track VST3 plugin changes.
- Use Open VST Plugin to open the selected track's applied VST3 editor.
- One VST plugin editor window is allowed per track.
- VST3 Settings controls Safe Plugin UI Mode, preferred plugin UI GPU, and maximum open plugin windows.
- Effects, MIDI tools, and unsupported plugins are not shown as track instruments unless manually enabled in Plugin Manager.
- Unknown plugins are hidden by default. Enable them only when you know they accept MIDI notes and produce audio.

VST3 Plugin Manager
-------------------

Open VST Plugins > VST3 Plugin Manager to review scanned plugins. The manager uses a dual-pane layout:

- Left pane: plugins not available as track instruments.
- Right pane: plugins available in the Instrument dropdown.

Use the filter dropdown and search box to narrow the list. Select one or more plugins and use the arrow buttons to move them between panes. Changes are pending until Apply Changes is clicked.

The picker panes use larger text, taller rows, and both vertical and horizontal scrolling. Long plugin names, vendors, categories, paths, and status text can be reviewed without relying on clipped rows.

The details panel shows metadata, paths, classification reasons, compatibility notes, and failure status for the highlighted plugin. If multiple plugins are selected, it shows a bulk-selection summary.

VST3 Settings And Plugin Windows
--------------------------------

VST3 Settings includes:

- Safe Plugin UI Mode.
- Preferred Plugin UI GPU. Saved GPU preferences are checked on startup/open so the selected adapter appears without requiring manual Refresh.
- Maximum open VST plugin windows, default 4, hard safety cap 12.
- Compatibility warning controls.

Plugin editor windows are edit/control surfaces. Piano Roll Preview and render handle note playback. Plugins usually do not play notes by themselves when you move controls; preview the track to hear the current state.

When a plugin editor is open, a small borderless host-controls palette appears with Apply Changes. The palette:

- is separate from the plugin UI,
- stays above the editor,
- uses the VST plugin window icon,
- can be dragged by click-holding anywhere on the palette, including over the button,
- can be moved outside the plugin window,
- closes automatically with the plugin editor.

After editing plugin controls, click Apply Changes so Piano Roll Preview and project rendering use the updated plugin state. Apply uses a compact one-button palette and a conservative deferred state-capture path. Normal note edits, Piano Roll undo/redo, preview, render, and closing the plugin editor do not auto-capture VST state blobs, which reduces stale native plugin state risk.

Full out-of-process crash isolation is reserved for the helper host in workspace\vst_host\PoorMansStudioVstHost.exe. Until then, VST3 plugin hosting remains in-process.

Tracks And Sequences
====================

MIDI Tracks
-----------

A MIDI track contains note events and uses an assigned instrument source. Supported MIDI sources are SF2/SF3, SFZ, and VST3 plugin instruments.

MIDI tracks are editable in the main window, Track Manager, and Piano Roll. Multiple Piano Roll windows can be open at once, but each track is limited to one Piano Roll window. Reopening a track that already has a Piano Roll open focuses that existing window.

AudioClip Tracks
----------------

An AudioClip track contains one imported or recorded audio file.

1 imported/recorded audio file = 1 AudioClip = 1 AudioClip track

AudioClip tracks use the built-in Custom Audio identity. They are managed like normal tracks in Track Manager and are included in preview and render paths.

Sequences
---------

Sequences group tracks for organization and preview/render workflows. The Track Manager and Change Track Seq picker stay synchronized with sequence membership and show empty sequences explicitly.

Track Manager
-------------

Track Manager edits track names, sequence placement, mute/solo, order, and related track organization. It prompts before closing when there are unapplied changes. Apply Changes commits the edit session. Discard and Exit abandons the edit session.

Piano Roll
==========

The Piano Roll edits MIDI notes for one track. It shows the current or pending instrument in the header and stages instrument/library changes until Apply Track Settings is clicked.

Common Piano Roll controls:

- Add, move, resize, and delete notes.
- Change default note length and velocity.
- Scroll the pitch view with the mouse wheel or scrollbar.
- Use note up/down controls for note-range steps.
- Adjust beat windows and pages to focus on smaller or larger sections.
- Preview the track to hear note playback.

Piano Roll windows preserve unsaved note edits and pending track settings while sequence membership changes are made elsewhere.

AudioClip Import And Recording
==============================

Import Audio supports WAV, MP3, FLAC, and OGG. Imported audio is copied or converted into the current project folder and stored as an AudioClip track. Saved AudioClip formats include WAV, FLAC, MP3, and OGG.

The AudioClip Recorder opens without auto-recording. Recorder behavior:

- Input device dropdown applies immediately.
- Refresh Devices rescans input devices.
- Mic Gain applies software recording gain before writing the WAV file.
- Record Test runs a 3-second countdown, records a 5-second temporary check, plays it back once, then deletes it.
- Record Test does not create an AudioClip track.
- Record starts after a short safety delay.
- Record With Delay supports 3, 5, 10, or custom delay up to 99 seconds.
- Pause pauses without writing a silent gap.
- Save / Apply commits the take as an AudioClip track.
- Discard Take cleans up temp audio.

Preview And Render
==================

Preview supports selected track, sequence, or full project playback, including AudioClip tracks. For VST3 tracks, Piano Roll Preview uses an independent render instance and the latest state intentionally committed with Apply Changes.

Render Settings include:

- output format,
- sample rate,
- bitrate,
- channel count,
- keep stem files,
- parallel stem renders.

Parallel Stem Renders can be Auto, 1, 2, 4, 6, 8, 12, or 16. This controls how many independent MIDI stem render jobs may run at the same time; it is not a literal CPU-core selector. Auto is the safest default.

Saving And Project Safety
=========================

Save Project writes the .mwproj metadata and project-relative paths. Start From File checks for unsaved project/editor changes before replacing the current project.

Dirty-change prompts use clear choices:

- Save As or Save and Exit when you want to keep the current project.
- Discard when you intentionally want to abandon changes.
- Cancel when you want to return to editing.

Troubleshooting
===============

No VST3 plugins appear
----------------------

Use VST Plugins > Scan VST3 Plugins. Confirm the plugin bundle is in workspace\vst3 or the standard Windows VST3 folder. Confirm the plugin is an instrument, not an effect or MIDI tool.

A plugin is hidden from the Instrument dropdown
-----------------------------------------------

Open VST3 Plugin Manager, select the plugin, and read the details panel. Effects and MIDI tools are not track instruments. Unknown plugins can be manually enabled only when you know they produce audio from MIDI notes.

A VST plugin crashes or fails to open
-------------------------------------

Save your project, reopen the app, and use VST3 Plugin Manager to clear or review failed/blocked state. Try Safe Plugin UI Mode. Avoid unfamiliar plugins in important projects until they are tested.

No sound from Piano Roll with VST3
----------------------------------

Make sure the track is assigned to a VST3 instrument plugin, not an effect. Open the Piano Roll, confirm the instrument header, then use Piano Roll Preview to send the track notes through the plugin.

Audio media is missing after moving a project
---------------------------------------------

Move or share the full project folder. The .mwproj file does not contain bulk audio media.

SFZ samples are missing
-----------------------

Keep the .sfz file and its same-name folder together. Check sample= and #include paths in the SFZ file. Use Test SFZ or render preflight messages to locate missing samples.
