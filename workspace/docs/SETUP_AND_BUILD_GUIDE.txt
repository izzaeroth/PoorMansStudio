Poor Man's Studio - Setup And Build Guide
==========================================

Current app version: 0.60.6

This guide describes the current source/build layout and runtime folder expectations. It documents current setup and runtime expectations only.

Build Environment
=================

Recommended Windows environment:

- Windows 10 or Windows 11, 64-bit.
- Visual Studio Community with Desktop development with C++.
- MSVC C++ x64/x86 build tools.
- Windows 10 or Windows 11 SDK.
- MSBuild.
- C++ CMake tools for Windows.
- CMake 3.24 or newer.
- Git for Windows, recommended for obtaining/updating JUCE.
- C++20-capable compiler.

JUCE
====

The source bundle expects JUCE at:

external\JUCE

JUCE is not bundled as a runtime dependency. Add the JUCE source tree before configuring the CMake project.

Configure And Build
===================

From a Visual Studio Developer PowerShell or normal PowerShell with CMake/MSBuild available:

cmake -S . -B build -G "Visual Studio 17 2022" -A x64
cmake --build build --config Release

The helper target PoorMansStudioVstHost is copied into workspace\vst_host by the local build. The main app expects its workspace folders beside the executable/source layout.

Workspace Runtime Notes
=======================

- workspace\exports is the default render/export destination.
- workspace\projects contains saved project folders.
- workspace\temp contains preview files, temporary renders, and active AudioClip staging sessions.
- workspace\vst3 contains portable/test VST3 bundles, snapshots, and the bundled test WAV under workspace\vst3\test.
- runtime temp/projects/settings folders are intentionally excluded from source bundles.

AudioClip Staging
=================

Imported and recorded AudioClip media is not embedded into .mwproj files. Unsaved imports and Save / Apply recorded takes stage in the active workspace\temp\recordings\rec_... session. Save Project moves attached staged imports into input\audio\imported and attached staged recordings into input\audio\recorded inside the saved project folder. Discarding unsaved changes from New/Open/Start From File/Exit prompts removes the active staged session so media cannot attach to the wrong later project.

VST3 Setup
==========

Use VST Plugins > Scan VST3 Plugins after adding or installing plugins. The Plugin Manager classifies plugins as Supported Instruments, Supported Effects, or Unsupported. Supported Effects populate both VST Effect dropdowns in the track inspector. Supported Instruments can be chosen through Change Library > Choose scanned VST3 plugin, which opens a full chooser dialog. Open plugin editors can be found again from Window > VST Instrument and Window > VST Effect side submenus.

Two-slot VST Effect Chain
=========================

Each track has two effect slots:

VST Effect 1: [dropdown] [Enable] [Bypass] [Open Slot 1]
VST Effect 2: [dropdown] [Enable] [Bypass] [Open Slot 2]

Processing order is source -> slot 1 -> slot 2 -> preview/export. Open Slot 1/2 is available only when that slot has an assigned plugin and its Enable checkbox is checked. Use Apply Track Settings after changing dropdowns, Enable, or Bypass. Use Apply Changes in the plugin editor after changing plugin parameters. Imported and recorded AudioClip source media stays dry; preview/export uses temporary processed WAVs.

The AudioClip Recorder Track Live Effect option is monitor-only. It uses the first active effect slot it can find on the target/source track for monitoring while keeping the recorded clip dry.

Troubleshooting Build Issues
============================

- If CMake cannot find JUCE, confirm external\JUCE exists and contains JUCE's CMake files.
- If the app builds but render/export fails, configure FluidSynth, FFmpeg, and sfizz-render runtime tools.
- If VST3 scanning fails, check the plugin bundle path and mark failed plugins unsupported until reviewed.
- If a plugin editor behaves badly, enable Safe Plugin UI Mode and lower the maximum open plugin-window count.
